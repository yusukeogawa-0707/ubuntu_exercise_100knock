# Git 100本ノック — 入門編

## 位置づけ

VSCode利用者でもCLIで履歴・差分・ブランチを読めるようにする。

- 入門編: まず壊さず、意味を確認しながら手を動かす。
- 実践編: 研究実務で使う形に近づける。

## 使い方

1. まず各Knockのコマンドを読む。
2. 練習用ディレクトリで実行する。
3. 解説と確認ポイントを見て、何が起きたか確認する。

注意: `myserver`, `your_username`, `git@example.com:USER/REPO.git` などは仮の名前です。実環境に合わせて置き換えてください。

## Knock 001: 練習用リポジトリを作る

**目的**: Git練習用の安全な場所を作る。

````bash
mkdir -p ~/git100knock && cd ~/git100knock && git init
````
**解説**: 実プロジェクトで試す前に、練習用repoで手を動かします。

**確認ポイント**: `.git/` ができる。


## Knock 002: ユーザ名を確認する

**目的**: Gitのコミット著者名を確認する。

````bash
git config --global user.name || true
````
**解説**: 未設定だとコミット時に警告が出ます。

**確認ポイント**: 名前または空表示。


## Knock 003: メールを確認する

**解説**: 会社用・個人用で混ざらないように注意します。

**確認ポイント**: メールまたは空表示。
