# Linux権限・トラブルシュート100本ノック — v2 応用・実践編

## 位置づけ

Permission deniedや容量・プロセス問題の初動確認を身につける。

- v1入門編: まず壊さず、意味を確認しながら手を動かす。
- v2応用・実践編: 研究実務で使う形に近づける。

## 使い方

1. まず各Knockのコマンドを読む。
2. 練習用ディレクトリで実行する。
3. 解説と確認ポイントを見て、何が起きたか確認する。

注意: `myserver`, `your_username`, `git@example.com:USER/REPO.git` などは仮の名前です。実環境に合わせて置き換えてください。

## Knock 001: 権限表示を見る

**目的**: ファイル権限の表示に慣れる。

````bash
ls -l
````
**解説**: 左端の `-rw-r--r--` のような部分が権限です。

**確認ポイント**: 権限・所有者・グループが見える。


## Knock 002: 自分のユーザを確認する

**目的**: 現在のユーザを確認する。

````bash
whoami
````
**解説**: Permission denied時は誰として操作しているかが重要です。

**確認ポイント**: ユーザ名が出る。


## Knock 003: 所属グループを見る

## Knock 004: ファイルを作る

## Knock 005: 権限を数値表示する

**目的**: 権限を記号と数値で見る。

````bash
stat -c "%A %a %U %G %n" sample.txt
````
**解説**: `644` のような数値権限を確認できます。

**確認ポイント**: 権限・所有者が出る。


## Knock 006: ディレクトリのx権限を理解する

**目的**: ディレクトリ権限の特殊性を見る。

````bash
mkdir -p dirx && echo secret > dirx/a.txt && chmod 600 dirx && ls dirx 2>&1 || true
````
**解説**: ディレクトリのxは「中を辿れる」権限です。

**確認ポイント**: lsできない場合がある。


## Knock 007: ディレクトリ権限を戻す

## Knock 008: 共有向けディレクトリを作る

**目的**: setgid付き共有ディレクトリを作る。

````bash

````
**解説**: `2` はディレクトリ配下にグループ継承を効かせる用途で使われます。

**確認ポイント**: 2775が表示される。

## Knock 010: umaskを一時変更する

**目的**: 新規ファイル権限への影響を見る。


**解説**: このシェルだけで影響します。戻すには新しいシェルが安全です。

**確認ポイント**: 664寄りの権限になる。
