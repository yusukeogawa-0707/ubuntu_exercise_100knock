# ネットワーク・SSH詳細100本ノック — v2 応用・実践編

## 位置づけ

VPN/SSH/HTTP/ポートフォワード/疎通確認を研究サーバ前提で扱う。

- v1入門編: まず壊さず、意味を確認しながら手を動かす。
- v2応用・実践編: 研究実務で使う形に近づける。

## 使い方

1. まず各Knockのコマンドを読む。
2. 練習用ディレクトリで実行する。
3. 解説と確認ポイントを見て、何が起きたか確認する。

注意: `myserver`, `your_username`, `git@example.com:USER/REPO.git` などは仮の名前です。実環境に合わせて置き換えてください。

## Knock 001: IPアドレスを見る

**目的**: ネットワークインターフェースとIPを見る。

````bash
ip addr
````
**解説**: VPN接続時の変化確認に使います。

**確認ポイント**: inet行を見る。


## Knock 002: IP行だけ抽出する

**目的**: 出力を絞る。

````bash
ip addr | grep "inet "
````
**解説**: ネットワーク情報は多いのでgrepで絞ります。

**確認ポイント**: IPアドレス行だけ出る。


## Knock 003: 経路を見る

**目的**: 通信経路を見る。

````bash
ip route
````
**解説**: VPNやサーバ接続トラブル時に重要です。

**確認ポイント**: default route等が出る。


## Knock 004: DNS確認する

**目的**: 名前解決できるか見る。

````bash
getent hosts example.com
````
**解説**: pingできなくてもDNS確認は別にできます。

**確認ポイント**: IPとホスト名が出る。


## Knock 005: curlで疎通確認

**目的**: HTTPヘッダだけ取得する。

````bash
curl -I https://example.com
````
**解説**: Web/API疎通確認の基本です。

**確認ポイント**: HTTPステータスが出る。


## Knock 006: Jupyter用ポートフォワードの型

**目的**: リモートJupyterを手元ブラウザで見る型を覚える。

````bash

````
**解説**: サーバ側localhost:8888をローカル8888へ転送します。

**確認ポイント**: 接続中はターミナルが占有される。

## Knock 008: バックグラウンドSSHトンネル

**目的**: SSHトンネルだけを裏で張る。

````bash
ssh -N -f -L 8888:localhost:8888 myserver
````
**解説**: `-N` はコマンドなし、`-f` はバックグラウンドです。

**確認ポイント**: psでsshプロセス確認。


## Knock 009: SSHトンネルプロセスを探す

**目的**: バックグラウンドトンネルを探す。

````bash
pgrep -af "ssh -N"
````
**解説**: 張ったトンネルを忘れないための確認です。

**確認ポイント**: 該当プロセスが出る。


## Knock 010: ポート転送中のローカルポート確認

**目的**: ローカルで待ち受けているか確認する。

````bash
ss -ltnp 2>/dev/null | grep ":8888" || true
````
**解説**: 転送が張れているかの確認です。

**確認ポイント**: 8888が出る場合がある。


## Knock 011: curlでローカル転送先確認

**目的**: 転送先サービスにHTTP確認する。

**確認ポイント**: HTTP応答または失敗。


## Knock 012: Proxy環境変数を見る

## Knock 013: curlで時間計測する

**目的**: HTTP応答時間を見る。

````bash
curl -L -o /dev/null -s -w "time_total=%{time_total}
http_code=%{http_code}
" https://example.com
````
**解説**: ネットワークが遅いのかサーバが遅いのか切り分けに使えます。

**確認ポイント**: 時間とstatusが出る。


## Knock 014: ダウンロード速度を見る

**目的**: ダウンロード速度を確認する。


**確認ポイント**: speedが出る。


## Knock 015: wgetで再開の型を見る
