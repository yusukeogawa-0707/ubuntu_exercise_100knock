# ネットワーク・SSH詳細100本ノック — 入門編

## 位置づけ

VPN/SSH/HTTP/ポートフォワード/疎通確認を研究サーバ前提で扱う。

- 入門編: まず壊さず、意味を確認しながら手を動かす。
- 実践編: 研究実務で使う形に近づける。

## 使い方

1. まず各Knockのコマンドを読む。
2. 練習用ディレクトリで実行する。
3. 解説と確認ポイントを見て、何が起きたか確認する。

注意: `myserver`, `your_username`, `git@example.com:USER/REPO.git` などは仮の名前です。実環境に合わせて置き換えてください。

## Knock 001: IPアドレスを見る

**目的**: ネットワークインターフェースとIPを見る。

````bash
ip addr
````
**解説**: VPN接続時の変化確認に使います。

**確認ポイント**: inet行を見る。


## Knock 002: IP行だけ抽出する

**目的**: 出力を絞る。

````bash
ip addr | grep "inet "
````
**解説**: ネットワーク情報は多いのでgrepで絞ります。

**確認ポイント**: IPアドレス行だけ出る。


## Knock 003: 経路を見る

**目的**: 通信経路を見る。


**解説**: VPNやサーバ接続トラブル時に重要です。

**確認ポイント**: default route等が出る。
